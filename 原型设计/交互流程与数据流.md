## 一、交互范式

**核心原则：用户通过自然语言驱动，不通过下拉框/表单选择。**

```
用户输入自然语言
       │
       ▼
┌──────────────────┐
│ Agent 理解意图    │ ← 从自然语言中提取：意图、品牌、参数
│ + 提取结构化参数  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 自动执行分析      │ ← 无需用户确认参数（Agent 自动推断）
│ （流式展示步骤）   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 输出结果          │ ← 分层卡片 / 文案变体 / 预测报告
└──────────────────┘
```

### 交互示例

| 用户说什么 | Agent 自动提取 | 触发场景 |
|-----------|---------------|---------|
| "帮我给百丽做会员分层，看近90天" | brand=百丽, period=90天 | 场景一 |
| "安踏高价值会员促活文案，要高端感" | brand=安踏, segment=高价值, goal=促活, tone=高端 | 场景二 |
| "预测满399减60效果，目标高价值，企微7天" | offer=满399减60, segment=高价值, channel=企微, duration=7 | 场景三 |
| "慕思最近半年复购率怎么样" | brand=慕思, period=180天, metric=复购率 | 自定义查询 |

```
┌──────────────────────────────────────────────────────┐
│                     Streamlit App                    │
├────────────┬─────────────────────────────────────────┤
│            │                                         │
│  侧边栏    │  🤖 Agent 对话区（主交互界面）          │
│  - 历史对话 │  ┌──────────────────────────────────┐  │
│  - 快捷入口 │  │ 💬 用户：帮我给百丽做会员分层…    │  │
│            │  │                                   │  │
│            │  │ 🧠 Agent：理解意图 → 提取参数 →   │  │
│            │  │     执行中… → 分层结果 + 策略卡片  │  │
│            │  └──────────────────────────────────┘  │
│            │                                         │
│            │  ┌──────────────────────────────────┐  │
│            │  │ 📝 输入框（自然语言）  [发送]     │  │
│            │  └──────────────────────────────────┘  │
└────────────┴─────────────────────────────────────────┘
```

## 二、场景一：会员分层策略生成

### 2.1 用户操作

| 步骤 | 用户操作 | 系统反馈 |
|------|---------|---------|
| 1 | 输入自然语言，如"帮我给百丽做会员分层，看近90天数据" | Agent 自动识别意图、品牌、周期 |
| 2 | 等待 Agent 执行（无需手动选择参数） | 5步执行流程流式展示 |
| 3 | 查看分层策略结果 | 分层卡片 + 策略建议 |

### 2.2 Agent 内部流转

```
用户点击「开始智能分层」
         │
         ▼
┌─────────────────┐
│ Step 1: 理解意图  │ ← understand_intent 节点
│ intent=segmentation│
│ brand=belle, days=90│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 2: 检索数据  │ ← retrieve_data 节点
│ Mock数据库查询    │
│ 返回 146,000 条   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 3: 计算RFM   │ ← generate_segmentation 节点
│ + K-Means聚类    │   内部调用 SegmentationTool
│ 6层，silhouette  │
│ = 0.72           │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 4: LLM策略  │ ← DeepSeek API 调用
│ DeepSeek V3      │   Prompt: 分层数据 + Few-shot
│ 生成差异化策略    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 5: 输出报告  │ ← generate_report 节点
│ JSON → 前端渲染  │
└─────────────────┘
```

### 2.3 数据接口

**前端 → 后端**

```json
POST /api/agent/chat
{
  "message": "帮我给百丽品牌做会员分层，分析近90天数据",
  "brand_id": "belle",
  "stream": true
}
```

**后端 → 前端（SSE 流式）**

```
event: step
data: {"step": 1, "status": "done", "label": "理解意图", "detail": "intent=segmentation"}

event: step
data: {"step": 2, "status": "done", "label": "检索数据", "detail": "拉取146,000条记录"}

event: step
data: {"step": 3, "status": "running", "label": "计算RFM", "detail": "K-Means聚类中..."}

event: step
data: {"step": 4, "status": "running", "label": "LLM策略生成", "detail": "DeepSeek推理中..."}

event: result
data: {"type": "segmentation", "segments": [...], "timestamp": "..."}

event: done
data: {}
```

## 三、场景二：个性化文案生成

### 3.1 用户操作

| 步骤 | 用户操作 | 系统反馈 |
|------|---------|---------|
| 1 | 输入自然语言，如"给高价值会员生成促活文案，品牌调性高端轻奢" | Agent 自动识别目标群体、活动目标、调性 |
| 2 | 等待 Agent 执行 | 向量检索→Few-shot生成→转化率预测 |
| 3 | 查看3版文案 + 转化率对比 | 推荐最优文案 |

### 3.2 Agent 内部流转

```
用户点击「生成文案变体」
         │
         ▼
┌──────────────────────┐
│ Step 1: 理解意图       │
│ intent=copywriting    │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 2: 向量检索       │
│ Chroma 召回 5 条       │
│ 历史高转化文案          │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 3: Few-shot生成   │
│ DeepSeek V3           │
│ 生成 3 版文案变体       │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 4: 转化率预测     │
│ 基于文案特征 + 历史    │
│ 输出 A→28% B→35% C→22%│
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 5: 推荐输出       │
│ 排序 + 推荐理由        │
└──────────────────────┘
```

### 3.3 数据接口

**前端 → 后端**

```json
POST /api/agent/chat
{
  "message": "给高价值会员生成促活文案，品牌调性高端轻奢",
  "brand_id": "belle",
  "context": {
    "target_segment": "高价值活跃会员",
    "campaign_goal": "促活",
    "brand_tone": "高端轻奢",
    "n_variants": 3
  },
  "stream": true
}
```

**后端 → 前端（SSE 流式，同上格式）**

## 四、场景三：活动效果预测

### 4.1 用户操作

| 步骤 | 用户操作 | 系统反馈 |
|------|---------|---------|
| 1 | 输入自然语言，如"预测满399减60活动效果，目标高价值会员，企微触达，周期7天" | Agent 自动识别优惠方案、人群、渠道、周期 |
| 2 | 等待 Agent 执行 | 双路检索→多维推理→基准对比 |
| 3 | 查看预测报告（6项指标 + 风险 + 建议） | 行业基准对比 + 优化建议 |

### 4.2 Agent 内部流转

```
用户点击「预测活动效果」
         │
         ▼
┌──────────────────────┐
│ Step 1: 理解意图       │
│ intent=prediction     │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 2: 双路检索       │
│ 向量检索(语义相似)     │
│ + 结构化查询(同品牌)   │
│ → 融合去重 Top 10      │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 3: LLM多维推理   │
│ DeepSeek V3           │
│ 预测 6 项指标          │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 4: 基准对比+风险  │
│ 行业基准加载           │
│ 阈值检查 → 风险预警    │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ Step 5: 优化建议输出   │
│ 规则+LLM混合建议       │
└──────────────────────┘
```

### 4.3 数据接口

**前端 → 后端**

```json
POST /api/agent/chat
{
  "message": "预测高价值会员满399减60活动效果，7天周期",
  "brand_id": "belle",
  "context": {
    "target_segment": "高价值活跃会员",
    "offer": {"type": "满减", "threshold": 399, "discount": 60},
    "channels": ["企微1v1", "企微群发"],
    "duration_days": 7
  },
  "stream": true
}
```

**后端 → 前端（SSE 流式）**

```

data: {"step": 1, "status": "done", "label": "理解意图"}

event: step
data: {"step": 2, "status": "done", "label": "检索历史活动", "detail": "双路召回10条"}
```
